# 提示词炼金炉 — 项目规划

> 开源工具：让不懂代码、不懂英语、不懂 AI 的普通人，通过飞书聊天把模糊想法炼成精准编程提示词，用于 Codex / Claude Code / Trae 等 AI 编程工具。

---

## 一、项目定义

```
输入：普通人一句模糊的中文（"帮我做个记账的"）
  ↓
Hermes 引导式对话（判断真伪需求、追问细节）
  ↓
输出：结构化中英双语精准提示词
  ↓
用户自行丢给 Codex / Claude Code / Trae 执行
```

**不是**：写代码的工具、SaaS 平台、商业产品
**是**：开源 AI 辅助需求分析工具，用户自己配 token，自己部署

---

## 二、核心设计原则

### 1. 信任 > 功能
普通人怕 AI、不信 AI、不知 AI 能干嘛。第一步不是分析需求，是建立信任。
- 开场白清晰自介："我是提示词助手，不是写代码的"
- 能力透明："我能做网页/小程序/脚本，造不了火箭"
- 所见即所得：每一步让用户看到进展

### 2. 对话质量 > 技术能力  
AI 成功率 60-80%，对普通人来说 60% = 废物。
- 降级策略：做不了全栈就做静态页，做不了部署就给 HTML
- 包装失败："这个有点复杂，我先帮你理清需求，你可以拿去让程序员接着做"
- 替代方案：主动建议"做不了 A，但类似的 B 我可以帮你"

### 3. 数据闭环
每次对话沉淀为"普通人如何描述技术需求"的语料。
- 统计高频需求类型
- 优化追问模板
- 积累"模糊描述 → 精准术语"映射表

---

## 三、对话流程（核心）

### 阶段 0：开场
```
用户首次对话 → Hermes 自我介绍
"你好！我可以帮你把想法变成 AI 编程工具能理解的描述。
  比如你想做个网站、小程序、或者自动化处理数据，
  跟我说说你想做什么？"
```

### 阶段 1：需求捕获（2-3 轮）
针对用户的模糊描述，分类追问：

| 维度 | 追问模板 |
|------|---------|
| 目的 | "这个东西用来做什么？谁用？" |
| 形式 | "你希望它是个网页、手机 App、还是小程序？" |
| 功能 | "它需要具备哪些功能？比如能不能登录、存数据？" |
| 外观 | "大概长什么样？有参考的 app 或网站吗？" |
| 数据 | "需要存储什么信息？数据从哪来？" |

### 阶段 2：真伪需求判断
- 真需求：技术可行、用户能说清
- 伪需求：技术不可行 → 告知边界 + 给替代方案
- 模糊需求：继续追问，直到能形成完整 prompt

### 阶段 3：补齐与翻译（核心能力）
```
用户原始输入 + 追问结果
  → 结构化需求分析（中文）
  → 技术术语映射（模糊词 → 精准词）
  → 多引擎 prompt 生成：
    - Codex 风格（强调项目结构、技术栈）
    - Claude Code 风格（强调交互式、迭代）
    - 通用风格（任何 AI 工具可用）
  → 中英双语输出
```

### 阶段 4：输出与交付
```
输出格式：
  📋 需求分析：
    你想做的是 [一句话概括]
    核心功能：[列表]
    技术方案建议：[建议]

  📝 编程提示词（中文）：
    [精准中文 prompt]

  📝 Programming Prompt (English)：
    [English prompt]

  🔧 下一步：
    - 用 Codex：复制英文 prompt 到 Codex CLI
    - 用 Claude Code：复制到 Claude Code 对话
    - 用 Trae：粘贴到 Trae 的 AI 对话
```

## 四、技术架构

```
用户（飞书）
    │
    ▼
飞书 Bot（Hermes Gateway）
    │
    ▼
Hermes Agent
    ├── skill: prompt-refinery（核心）— 引导对话 + 需求炼化
    ├── skill: requirement-classifier — 真伪需求判断
    ├── skill: tech-mapper — 模糊词 → 技术术语映射
    ├── skill: prompt-builder — 多引擎 prompt 模板生成
    └── memory — 用户偏好、历史项目
```

---

## 五、交付物

### 用户侧
1. **安装脚本** — 一键部署 Hermes + 飞书配置
2. **飞书 Bot 配置指南** — 教用户注册飞书 App、获取 token
3. **使用说明** — "我能做什么 / 不能做什么" 清单

### 开发者侧
1. **Hermes 技能包**（4 个 SKILL.md）
2. **配置文件模板**（config.yaml、.env 模板）
3. **示例对话** — 3 个完整引导案例
