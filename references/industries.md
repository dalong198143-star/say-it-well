每个画像包含：典型场景、核心指标、常见痛点、追问方向、技术栈建议

## 1. 餐饮
- **典型场景**: 堂食点单、外卖接单、后厨出餐、排队叫号、会员储值
- **核心指标**: 翻台率、客单价、平台抽成(20-25%)、食材成本占比(30-35%)
- **常见痛点**: 高峰排队流失、外卖平台抽成高、后厨出餐慢、会员复购低
- **追问方向**: 堂食还是外卖为主？几桌/几连锁？要不要扫码点餐？后厨有没有 KDS 屏？
- **技术栈**: 微信小程序 + 云开发（小餐饮）/ React + Node.js + MySQL（连锁）

## 2. 教育 / 培训
- **典型场景**: 排课选课、学员管理、作业批改、直播授课、续费提醒
- **核心指标**: 续费率（K12 看续报、成人看完课率）、排课利用率、获客成本
- **常见痛点**: 排课冲突、学员流失无预警、多校区数据不通、老师绩效难算
- **追问方向**: K12 辅导还是成人培训？线上还是线下？要不要排课自动检测冲突？
- **技术栈**: 微信小程序 + 腾讯云 TRTC（直播课）/ React + Django + PostgreSQL（后台）

## 3. 游戏陪玩 / 电竞
- **典型场景**: 陪玩匹配、段位筛选、接单派单、语音沟通、结算分账
- **核心指标**: 匹配效率（几秒内响应）、接单率、抽成比例（平台抽 20-30%）
- **常见痛点**: 派单人工效率低、扣 1 先到先得但质量参差、陪玩师排班混乱
- **追问方向**: 小程序下单还是企业微信？派单是先到先得还是按匹配度？接单后谁确认？
- **技术栈**: 微信小程序 + 云开发（前端）/ FastAPI + MySQL（后端）

## 4. 电商 / 零售
- **典型场景**: 商品浏览、购物车、下单付款、库存管理、物流追踪
- **核心指标**: 转化率、客单价、退换货率、库存周转天数
- **常见痛点**: 多平台库存不同步、优惠券规则复杂、订单高峰期发货慢
- **追问方向**: 自营卖货还是平台撮合？主要卖什么品类？有没有实体店？
- **技术栈**: 微信小程序 + 云开发 / React + Node.js + MongoDB（跨境电商）

## 5. 物流配送
- **典型场景**: 订单调度、路线规划、骑手/无人机管理、电子面单、签收确认
- **核心指标**: 配送时效（分钟级/小时级）、调度效率、妥投率
- **常见痛点**: 高峰期运力不足、路线规划不合理、签收纠纷
- **追问方向**: 即时配送还是同城？骑手还是无人机？要不要路线自动优化？
- **技术栈**: React 大屏 + FastAPI + PostgreSQL + 百度/高德地图 API

## 6. IT / 技术自媒体
- **典型场景**: 内容创作（博客/视频/播客）、知识付费、社群运营、工具开发
- **核心指标**: 内容产出频率、粉丝增长率、付费转化率
- **常见痛点**: 内容生产效率低、多平台分发麻烦、粉丝互动管理
- **追问方向**: 文字还是视频还是播客？自己用还是做产品给别人用？要不要 AI 辅助生成？
- **技术栈**: Next.js + MDX（博客）/ Flutter（App）/ Python FastAPI + OpenAI（AI 工具）

## 7. 医疗健康
- **典型场景**: 预约挂号、电子病历、在线问诊、慢病管理、体检报告
- **核心指标**: 预约利用率、问诊转化率、患者留存率
- **常见痛点**: 排队时间长、病历跨院不通、随访缺失
- **追问方向**: 诊所还是医院？面向患者还是内部管理？要不要对接 HIS 系统？
- ⚠️ 涉及医疗数据需《个人信息保护法》合规，等保三级
- **技术栈**: 微信小程序 + Java Spring Boot + MySQL（安全要求高）

## 8. 房地产
- **典型场景**: 房源管理、VR 看房、客户跟进、合同签约、佣金计算
- **核心指标**: 带看转化率、成交周期、房源去化率
- **常见痛点**: 房源信息更新不及时、客户跟丢、多中介撞单
- **追问方向**: 新房代理还是二手房？租赁还是买卖？多门店还是单店？
- **技术栈**: 微信小程序 + React Admin + Node.js + PostgreSQL

## 9. 金融 / 保险
- **典型场景**: 客户管理、产品推荐、理赔流程、风险评估、合规审批
- **核心指标**: 转化率、续保率、理赔时效
- **常见痛点**: 产品太多客户选不动、理赔流程长、合规要求严
- **追问方向**: 面向 C 端客户还是内部业务员？保险还是理财？要不要对接银保监会？
- ⚠️ 金融类涉及牌照和合规，AI 只能做辅助工具
- **技术栈**: Java Spring Boot + Oracle/MySQL（银行级安全）

## 10. 美容美发
- **典型场景**: 预约排班、会员储值、服务记录、产品库存、员工提成
- **核心指标**: 预约到店率、客单价、会员复购率、发型师人效
- **常见痛点**: 预约管理靠微信、会员流失无预警、员工提成算错
- **追问方向**: 单店还是连锁？要不要在线预约和储值？发型师提成怎么算？
- **技术栈**: 微信小程序 + 云开发（单店）/ React + Node.js（连锁）

## 11. 健身 / 运动
- **典型场景**: 会员管理、课程预约、私教排课、入场签到、体测记录、社区打卡
- **核心指标**: 会员活跃率、私教课时利用率、续卡率、体测数据增长率
- **常见痛点**: 会员办了卡不来、私教排课冲突、体测数据丢失、缺乏社区氛围
- **追问方向**: 健身房还是在家练？要不要体测数据追踪？要不要社区打卡互相激励？要不要对接智能门禁？
- **技术栈**: 微信小程序 + 云开发 / Flutter（跨平台 App）

## 12. 家政服务
- **典型场景**: 阿姨/师傅匹配、服务预约、评价体系、工资结算、背景调查
- **核心指标**: 匹配成功率、服务满意度、阿姨留存率、投诉率
- **常见痛点**: 档期难管、服务质量参差、工资结算纠纷、安全信任问题
- **追问方向**: 月嫂/保洁/维修哪一类？平台撮合还是自营？要不要背景调查和保险？
- **技术栈**: 微信小程序 + FastAPI + PostgreSQL

## 13. 旅游 / 酒店
- **典型场景**: 房间预订、入住退房、房态管理、评价管理、OTA 对接
- **核心指标**: 入住率、RevPAR、OTA 佣金占比(15-25%)
- **常见痛点**: 多平台房态不同步、旺季超卖、前台效率低
- **追问方向**: 酒店还是民宿？单体还是连锁？要不要对接携程/美团？
- **技术栈**: React + Node.js + MySQL / 微信小程序 + 云开发（民宿）

## 14. 汽车 / 出行
- **典型场景**: 车辆调度、司机管理、计费结算、维修保养、充电桩管理
- **核心指标**: 车辆利用率、司机接单率、客单价、维保成本
- **常见痛点**: 高峰运力调度难、司机考核公平性、维修保养忘记提醒
- **追问方向**: 网约车还是租车？新能源要不要充电桩管理？要不要维修提醒？
- **技术栈**: React + Java Spring Boot + PostgreSQL + 高德地图 API

## 15. 法律咨询
- **典型场景**: 案件管理、合同审查、法规查询、客户沟通、文书生成
- **核心指标**: 案件周转天数、合同审查效率、客户满意度
- **常见痛点**: 法律文书重复劳动多、法规更新跟不上、案件进度不透明
- **追问方向**: 律所还是企业法务？主要处理什么类型？要不要对接裁判文书网？
- ⚠️ AI 生成法律文书不具法律效力，需律师审核
- **技术栈**: Django + PostgreSQL（内网部署优先）

## 16. 制造业 / 工厂
- **典型场景**: 生产排程、工单管理、质检记录、设备维保、库存物料、工人计件
- **核心指标**: OEE（设备综合效率）、良品率、生产周期、物料周转
- **常见痛点**: 排产靠 Excel、质检数据丢失、设备故障无预警、工人计件工资纠纷
- **追问方向**: 离散制造还是流程制造？要不要对接 ERP？要不要设备 IoT 数据采集？工人计件还是计时？
- **技术栈**: React + Java Spring Boot + MySQL / 工业 IoT 平台（设备对接）

## 17. 农业
- **典型场景**: 种植管理、施肥打药、采摘排期、产量预测、溯源追踪、农机调度
- **核心指标**: 亩产量、病虫害发生率、采摘效率、农资成本
- **常见痛点**: 种植靠经验、病虫害发现晚、采摘人手调度、农产品溯源难
- **追问方向**: 种植还是养殖？大棚还是露地？要不要物联网传感器？要不要溯源到消费者？
- **技术栈**: 微信小程序 + 云开发（轻量）/ Django + PostgreSQL + IoT 平台

## 18. 文化传媒 / 娱乐
- **典型场景**: 演出票务、艺人管理、内容审核、版权管理、粉丝运营
- **核心指标**: 票房/票务转化率、内容产出量、粉丝增长率、版权收入
- **常见痛点**: 票务黄牛、内容审核慢、版权纠纷、粉丝运营工具少
- **追问方向**: 演出票务还是内容平台？面向 B 端还是 C 端？要不要实名制购票？
- **技术栈**: React + Node.js + MongoDB / 微信小程序

## 19. 宠物服务
- **典型场景**: 寄养预约、美容洗澡、医疗档案、宠物粮商城、遛狗匹配
- **核心指标**: 预约到店率、客单价、复购率、宠物健康档案完整度
- **常见痛点**: 预约靠微信、医疗档案丢失、宠物粮复购忘提醒
- **追问方向**: 单一服务还是综合平台？要不要对接宠物医院？要不要宠物粮定期配送？
- **技术栈**: 微信小程序 + 云开发

## 20. 婚庆 / 活动策划
- **典型场景**: 档期管理、方案设计、供应商协调、预算控制、客户沟通
- **核心指标**: 签单转化率、档期利用率、客户满意度、毛利率
- **常见痛点**: 档期冲突、供应商不靠谱、客户沟通成本高、预算超支
- **追问方向**: 婚庆为主还是企业活动？要不要 3D 方案预览？要不要供应商评分体系？
- **技术栈**: React + Django + PostgreSQL

## 21. 二手 / 闲置交易
- **典型场景**: 商品发布、搜索筛选、聊天议价、信用体系、物流对接
- **核心指标**: 发布量、成交率、客单价、纠纷率
- **常见痛点**: 假货泛滥、恶意议价、物流纠纷、信用体系缺失
- **追问方向**: C2C 还是 C2B2C？什么品类为主？要不要验货担保？要不要同城面交？
- **技术栈**: 微信小程序 + Node.js + MongoDB + 腾讯云 IM

## 22. 心理咨询
- **典型场景**: 咨询师匹配、预约排班、在线咨询（文字/语音/视频）、测评量表
- **核心指标**: 咨询师接单率、来访者留存率、测评完成率、满意度
- **常见痛点**: 匹配不精准、隐私保护要求高、危机干预机制缺失
- **追问方向**: 面向个人还是企业 EAP？要不要在线视频咨询？要不要心理测评量表？
- ⚠️ 心理咨询涉及《精神卫生法》，需持证上岗
- **技术栈**: 微信小程序 + Django + PostgreSQL（数据加密存储）

## 23. 装修 / 建筑
- **典型场景**: 设计报价、施工管理、材料采购、进度追踪、验收结算
- **核心指标**: 签单转化率、工期达成率、材料损耗率、客户满意度
- **常见痛点**: 报价不透明、施工进度不透明、材料浪费、验收纠纷
- **追问方向**: 家装还是工装？设计+施工全包还是只做管理？要不要工地直播？
- **技术栈**: 微信小程序 + React Admin + Node.js + PostgreSQL

## 24. 政府 / 公共服务
- **典型场景**: 办事预约、信息公开、投诉建议、数据大屏、内部 OA
- **核心指标**: 办事效率、群众满意度、信息公开率
- **常见痛点**: 流程繁琐、信息不透明、多部门数据不通
- **追问方向**: 面向公众还是内部办公？要不要对接现有政务系统？要不要数据大屏？
- ⚠️ 政务系统需等保三级、国产化适配（信创）
- **技术栈**: Java Spring Boot + 达梦/人大金仓数据库 + 国产中间件

## 25. 跨境电商
- **典型场景**: 多语言商品管理、跨境物流、关税计算、多币种支付、海外仓
- **核心指标**: GMV、客单价、物流时效、关税成本占比
- **常见痛点**: 多平台库存同步、物流时效不可控、汇率波动、各国合规不同
- **追问方向**: B2C 还是 B2B？主要目标市场？要不要对接 Amazon/Shopify/Lazada？
- ⚠️ 涉及跨境数据需注意 GDPR 等各国隐私法规
- **技术栈**: React + Node.js + PostgreSQL / Shopify App（轻量）
