# IMA Skill Cloning — 判断标准

## 克隆决策流程

```
看到IMA技能 → 三步判断：

1. 已有等价？
   ✅ 有 → 不克隆，告知已有（如：多引擎检索→AnySearch、语感润色→humanizer-zh）
   ❌ 无 → 继续

2. 技术上可做？
   ✅ 可 → 继续
   ❌ 不可 → 不克隆，说清原因（需要实时行情API/专业法律库/平台登录态等）

3. 有实用价值？
   ✅ 有 → 克隆
   ❌ 无 → 不克隆，给出理由
```

## 常见拒绝理由模板

- 「已有等价——xxx 已覆盖」
- 「需要实时数据/专业库/API认证——不适合轻量克隆」
- 「纯方法论模板——价值有限，不建独立工具」

## 反例（2026-05-26）

「AI社交人脉经营策略」→ 我说"不克隆"，没给理由。用户追问"为什么不能"。
实际判断：有实用价值（say-it-well 用户是自媒体创作者，需要社交策略）。
结论：不能因为"不是代码工具"就拒绝。

## 核心原则

判断标准是「有没有实用价值」，不是「技术复杂度高低」。
每次拒绝必须给明确理由。
